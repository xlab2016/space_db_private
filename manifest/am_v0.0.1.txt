Примитивы:

1) Структурно-семантический диск (SSD-structural semantic disk).

Структурно-семантический диск представляет структуру любой файловой системы в такую которую понимает интеллект. Основа структурно-семантической сети в том что структура работает поверх семантики. Структура задает графовую конфигурацию между вершинами. Каждая вершина представляет собой логический элемент: каталог, файл, логическая область файлов (например диск или компьютер), логические разделы файла, семантические разделы. Каждая вершина представляет собой структурный элемент, и содержит связи между собой. Вершина имеет координаты в латентном пространстве, которое представляет собой отражение энергии необходимого логического представления. Например в семантическом представлении пространства, вершины близкие по смыслу тематики располагаются рядом между собой и т.д.
Для организации представления семантического диска требуется структурно-семантическая файловая система: SSFS - structural semantic file system.

SSFS:
Structural layer
--logical spaces
----vertices, relations
Semantic layer
--vertex, relation
----context vector (position)=parent context vector+vector
----vector (position)
----type: structure, content
----data: any information
Ontology layer
----class scopes
----classes
------attributes, methods, rules etc.
----object scopes<class>
------objects

2) Структурно-семантический компилятор (SSC - structural semantic compiler). Это программный комплекс, реализуемый в драйвере структурно семантического диска. Он получает любую структуру файловой системы ОС, например каталог или сжатый архив каталогов, и производит компиляцию в структуру структурно-семантической файловой системы (SSFS). Интерпретатор исполнимой части AGI такой как представляемый язык программирования (Meta AGI Compiler), использует семантический диск для решения прикладных задач. В данном документе ниже представлен пример кода для низкоуровнего вызова компилятора SSC семантического диска. 

1) Sync version:
var stream1 := stream<file>;
stream1.open("sample_path");

var data = await stream1;

var semantic_file_system = compile(data);

2) Async version:
var stream1 := stream<file>;
stream1.open("sample_path");

for streamwait by delta (stream1, delta, aggregate) {
	var semantic_file_system_delta = await compile(delta);
}

3) Языки исполнения среды AGI.
Описание языка Meta AGI Compiler (MEAC).
Язык содержит синтаксис похожий на язык программирования Pascal, но семантически представляет собой язык программирования нового уровня, более высокоуровневый чем предшествующие языки.
Оператор <:=>. Этот оператор представляет собой не присваивание как в Pascal, а стриминг потока. В текущем примере:
var stream1 := stream<file>;
Означает стриминг потока через переменную stream1;
Генерализация классов, примитивный класс. Язык содержит классы примитивы (типы) которые являются ключевыми словами в языке. Язык сам контекстно понимает когда следует интерпретировать лексему как класс примитив. В текущем примере имеется следующая конструкция:
stream<file>
Это представляет собой декларацию класса примитива stream, но через генерализацию другого класса примитива file. Генерализация мутирует и суживает или расширяет семантику класса примитива. Например:
stream<file>: сужение.
stream<video, audio>: расширение.
Конструкция await/streamwait. Ключевое слово await дает команду интерпретатору или специализированной операционной системе команду входа в асинхронный режим. В этом режиме начинают асинхронно работать не только процессы, но и потоки данных. Асинхронный режим потока данных означает что в этом режиме поток обрабатывается интерпретатором или ОС (ИОС) считывая данные и обрабатывая полностью в асинхронном режиме и полностью поточно. Этот режим обеспечивает максимальную экономию памяти поскольку данные с потока считываются ограниченными порциями размер их управляется ИОС и при малой нагрузке повышается чтобы поднять пропускную способность системы, при большой понижается. Но этот режим требует от обработчиков данных способность работы в поточном режиме. Например для SSC компилятора диска это является необходимым условием.
Здесь var data = await stream1; задает поточный режим передачи данных и ИОС решает в случае если конечное устройство: var semantic_file_system = compile(data); здесь compile встроенное обращение к SSC текущего диска, поддерживает поточную обработку выполняет все в поточном режиме. В противном случае входит в специальный защищенный режим который пытается прочитать данные до допустимого предела и передать обработчику либо ИОС уведомляет пользователя о нехватке ресурса и невозможности выполнить execution unit. В этом случае, он может продолжиться когда освободятся ресурсы либо по требованию пользователя.
for streamwait by delta (stream1, delta, aggregate) - это конструкция итерирования по потоку, каждый цикл это считывание порции потока который попадает в переменную delta. delta aggregate это контекстные переменные и работают только внутри цикла. Их можно генерализировать в переменные, например так: for streamwait by delta (stream1, delta as delta1, aggregate as aggregate1). В случае вложенности итераторов. Итераторы потока обеспечивают просто доступ к устройствам имеющих поточную работу. Например к устройству LLM ввода/вывода (LLM IO).
